ODPS SQL

Chapter 2 基本语法

2.1 运算符

在 ODPS SQL 中，运算符用于执行各种数据操作，包括算术运算、比较运算、逻辑运算等。合理使用运算符是进行数据分析和处理的基础。

2.1.1 算术运算符

算术运算符用于执行基本的数学运算，包括加法、减法、乘法、除法和取余。

  1. 加法（+）：计算两个数值的和。
  2. 减法（-）：计算两个数值的差。
  3. 乘法（*）：计算两个数值的积。
  4. 除法（/）：计算两个数值的商。
  5. 取余（%）：计算两个数值相除后的余数。

2.1.2 比较运算符

比较运算符用于比较两个表达式的值，结果为布尔值（TRUE 或 FALSE）。

  1. 等于（=）：判断两个表达式是否相等。
  2. 不等于（<> 或 !=）：判断两个表达式是否不相等。
  3. 大于（>）：判断左边的表达式是否大于右边的表达式。
  4. 小于（<）：判断左边的表达式是否小于右边的表达式。
  5. 大于等于（>=）：判断左边的表达式是否大于或等于右边的表达式。
  6. 小于等于（<=）：判断左边的表达式是否小于或等于右边的表达式。

2.1.3 逻辑运算符

逻辑运算符用于连接多个布尔表达式，结果也为布尔值。

  1. 与（AND）：如果所有表达式都为真，则结果为真。
  2. 或（OR）：如果至少有一个表达式为真，则结果为真。
  3. 非（NOT）：如果表达式为假，则结果为真；如果表达式为真，则结果为假。

2.1.4 位运算符

位运算符用于对二进制位进行操作，包括按位与、按位或、按位异或、按位取反、左移和右移。

  1. 按位与（&）：对两个数的二进制表示进行逐位与操作。
  2. 按位或（|）：对两个数的二进制表示进行逐位或操作。
  3. 按位异或（^）：对两个数的二进制表示进行逐位异或操作。
  4. 按位取反（~）：对一个数的二进制表示进行逐位取反操作。
  5. 左移（<<）：将一个数的二进制表示向左移动指定的位数。
  6. 右移（>>）：将一个数的二进制表示向右移动指定的位数。
SELECT 6 & 3 AS bitwise_and,
       6 | 3 AS bitwise_or,
       6 ^ 3 AS bitwise_xor,
       ~6 AS bitwise_not,
       6 << 1 AS left_shift,
       6 >> 1 AS right_shift;

2.1.5 其他运算符

ODPS SQL 还支持其他一些运算符，用于特定场景下的数据操作，例如：

  1. 字符串连接运算符（||）：用于连接两个或多个字符串。
  2. 空值判断运算符（IS NULL、IS NOT NULL）：用于判断表达式是否为 NULL。
  3. 区间运算符（BETWEEN … AND …）：用于判断表达式的值是否在指定的区间内。
  4. 成员运算符（IN、NOT IN）：用于判断表达式的值是否在指定的集合中。

2.2 转义字符

转义字符在 ODPS SQL 中扮演着重要的角色，它们使得特殊字符在字符串中的使用成为可能。使用转义字符，可以在字符串中插入通常有特殊含义的字符，比如换行符、制表符等。

2.2.1 转义字符列表

以下是 ODPS SQL 中常见的转义字符及其用途：

  1. 退格（backspace），将当前位置退回到前一列。
  2. 水平制表符（tab），将当前位置移至下一个制表位。
  3. ：换行符（newline），将当前位置移到下一行开头。
  4. 回车（carriage-return），将当前位置移到本行开头。
  5. '：单引号，用于在字符串中表示单引号字符。
  6. "：双引号，用于在字符串中表示双引号字符。
  7. \：反斜杠，用于在字符串中表示反斜杠字符。
  8. ;：分号，用于在字符串中表示分号字符。
  9. ：表示 Control-Z，文件结束符。
-- 创建表并插入数据
CREATE TABLE IF NOT EXISTS escape_characters (
    example STRING
);

INSERT INTO escape_characters (example) VALUES
('First line.\nSecond line.'),
('Tab\tseparated'),
('Single quote: \''),
('Double quote: \"'),
('Backslash: \\'),
('Semicolon: \;'),
('End of file marker: \Z');

-- 查询数据
SELECT example
FROM escape_characters;

2.2.2 转义字符的重要性

在处理文本数据时，了解并正确使用转义字符至关重要。它们能够帮助用户处理如下情况：

  1. 字符串中的控制字符：当字符串中需要包含换行、制表等控制字符时。
  2. 特殊字符的字面量：当字符串中需要包含如单引号、双引号等通常具有特殊用途的字符时。
  3. 文本格式化：使用转义字符进行文本输出的格式化，例如，创建具有特定列间距的报表。

2.2.3 注意事项

  1. 转义序列的正确性：确保使用正确的转义序列，错误的使用可能会导致查询错误或意外的结果。
  2. 语言差异：不同的 SQL 实现可能对转义字符有不同的处理方式，需要根据使用的具体 SQL 方言调整转义序列。

2.3 LIKE 匹配符

LIKE 是 ODPS SQL 中使用的一个匹配符，用于在 WHERE 子句中搜索列中的特定模式。这个操作符通常与 %（代表任意字符序列）和 _（代表单个字符）一起使用。

2.3.1 LIKE 匹配符的使用

LIKE 操作符用于基于模式的匹配。其中，% 和 _ 是两个特殊的通配符：

  1. %：代表零个、一个或多个字符。
  2. _：代表一个单字符。

2.3.2 示例

假设我们有一个商品信息表 products，包含字段 product_name 和 product_code。

-- 创建商品信息表
CREATE TABLE IF NOT EXISTS products (
    product_name STRING,
    product_code STRING
);

-- 插入商品数据
INSERT INTO products (product_name, product_code) VALUES
('Special Cereal', 'CER001'),
('Special Cheese', 'CHE002'),
('Regular Bread', 'BRD003'),
('Regular Milk', 'MLK004');

如果我们想要找到所有名称中包含 “Special” 的商品，可以使用 LIKE：

-- 查询所有包含'Special'的商品
SELECT *
FROM products
WHERE product_name LIKE '%Special%';

2.3.3 LIKE 的高级用法

LIKE 不仅可以进行简单的包含搜索，还可以进行更复杂的模式匹配。例如，如果我们想要找到商品代码以 ‘CER’ 开头的商品，可以这样做：

-- 查询商品代码以'CER'开头的商品
SELECT *
FROM products
WHERE product_code LIKE 'CER%';

如果我们需要找到第三位字符为 ‘E’ 的商品名称，可以使用 _ 通配符：

-- 查询商品名称第三位字符为'E'的商品
SELECT *
FROM products
WHERE product_name LIKE '__E%';

2.3.4 使用 LIKE 时的注意事项

  1. 性能：LIKE 操作可能比其他类型的条件更慢，因为它通常无法利用索引。
  2. 匹配模式：匹配模式越具体，查询通常越快。例如，模式 ‘CER%’ 比 ‘%CER%’ 快，因为后者需要全表扫描。
  3. 转义字符：在使用 LIKE 时，如果需要搜索 % 或 _ 本身，需要使用转义字符。

2.3.5 转义字符在 LIKE 中的应用

如果需要在 LIKE 查询中查找 % 或 _，可以使用转义字符。默认情况下，LIKE 子句中的转义字符是 ，但也可以指定其他字符。

假设我们需要找到商品名称中包含 _ 的商品：

-- 查询商品名称中包含'_'的商品
SELECT *
FROM products
WHERE product_name LIKE '%\_%' ESCAPE '\';

2.4 RLIKE 匹配符

RLIKE（也称作 REGEXP），是 ODPS SQL 中用于实现正则表达式匹配的操作符。与 LIKE 相比，RLIKE 提供了更为强大和灵活的模式匹配能力。

2.4.1 RLIKE 匹配符的使用

RLIKE 允许使用正则表达式来定义匹配模式，这些模式可以包含一个或多个正则表达式特殊字符和构造，例如：

  1. .：匹配除换行符以外的任意单个字符。
  2. ^：匹配输入字符串的开始位置。
  3. $：匹配输入字符串的结束位置。
  4. *：匹配前面的子表达式零次或多次。
  5. +：匹配前面的子表达式一次或多次。
  6. ?：匹配前面的子表达式零次或一次。
  7. {n}：匹配确定的 n 次，n 是一个非负整数。
  8. {n,}：至少匹配 n 次。
  9. {n,m}：至少匹配 n 次且不超过 m 次。
  10. [xyz]：字符集合。匹配所包含的任意一个字符。
  11. [^xyz]：负值字符集合。匹配未包含的任意字符。
  12. [a-z]：字符范围。匹配指定范围内的任意字符。
  13. [^a-z]：负值字符范围。匹配不在指定范围内的任意字符。
  14. ：将下一个字符标记为特殊字符或字面值。
  15. |：选择。匹配 | 前后的任意子表达式。
  16. ()：标记一个子表达式的开始和结束位置。
-- 查询商品代码中包含数字'001'的商品
SELECT *
FROM products
WHERE product_code RLIKE '001$';

-- 查询商品名称以'Special'开头的商品
SELECT *
FROM products
WHERE product_name RLIKE '^Special';

-- 查询商品名称中含有任意数字的商品
SELECT *
FROM products
WHERE product_name RLIKE '[0-9]';

select 'aa123bb' rlike '^a';
-- 匹配字符串aa123bb的开头是否为a，返回true

select 'aa123bb' rlike '^a.*b$';
-- 匹配字符串aa123bb的开头是否为a、结尾是否为b，运行返回true

select 'footerbar' rlike 'foo(.*?)(bar)';
-- 返回true

2.4.2 RLIKE 高级元字符

RLIKE 还支持更复杂的正则表达式规则，包括字符类、预定义字符类、边界匹配符等。以下是一些高级元字符的例子：

  1. 匹配任何数字，等价于 [0-9]。
  2. ：匹配任何非数字字符，等价于 [^0-9]。
  3. ：匹配任何单词字符，包括下划线，等价于 [A-Za-z0-9_]。
  4. ：匹配任何非单词字符，等价于 [^A-Za-z0-9_]。
  5. ：匹配任何空白字符，包括空格、制表符、换页符等，等价于 [ 。
  6. ：匹配任何非空白字符，等价于 [^ 。
  7. 匹配单词边界。
  8. ：匹配非单词边界。

假设我们想找到商品代码中包含至少一个数字的所有商品：

-- 查询商品代码中包含至少一个数字的商品
SELECT *
FROM products
WHERE product_code RLIKE '\\d+';

2.5 ODPS SQL 保留字与关键字

在 ODPS SQL 中，保留字和关键字是预先定义的标识符，它们具有特殊的意义。关键字通常是 SQL 语法的一部分，而保留字是为系统功能保留的词汇。使用这些词汇作为表名、列名或别名时，可能需要使用特定的引用规则。

2.5.1 算术运算符

  • %：取模运算符，返回两个数相除的余数。
    • 示例：SELECT 10 % 3; -- 结果为 1
  • +：加法运算符。
    • 示例：SELECT 2 + 3; -- 结果为 5
  • -：减法运算符。
    • 示例：SELECT 5 - 2; -- 结果为 3
  • *：乘法运算符。
    • 示例：SELECT 3 * 4; -- 结果为 12
  • /：除法运算符。
    • 示例：SELECT 10 / 2; -- 结果为 5

2.5.2 比较运算符

  • =：等于运算符。
    • 示例：SELECT * FROM table WHERE column = 10;
  • <> 或 !=：不等于运算符。
    • 示例：SELECT * FROM table WHERE column <> 10;
  • <：小于运算符。
    • 示例：SELECT * FROM table WHERE column < 10;
  • <=：小于等于运算符。
    • 示例：SELECT * FROM table WHERE column <= 10;
  • >：大于运算符。
    • 示例：SELECT * FROM table WHERE column > 10;
  • >=：大于等于运算符。
    • 示例：SELECT * FROM table WHERE column >= 10;

2.5.3 逻辑运算符

  • AND 或 &&：逻辑与运算符。
    • 示例：SELECT * FROM table WHERE column1 = 10 AND column2 = 20;
  • OR：逻辑或运算符。
    • 示例：SELECT * FROM table WHERE column1 = 10 OR column2 = 20;
  • NOT：逻辑非运算符。
    • 示例：SELECT * FROM table WHERE NOT column = 10;

2.5.4 数据类型

  • BIGINT：64位整数类型。
  • BOOLEAN：布尔类型，值为 TRUE 或 FALSE。
  • DOUBLE：双精度浮点数类型。
  • STRING：字符串类型。
  • DECIMAL：定点数类型，用于表示精确的数值。

2.5.5 数据定义语言 (DDL)

  • CREATE：用于创建数据库对象，如表、视图等。
    • 示例：CREATE TABLE table_name (column1 INT, column2 STRING);
  • ALTER：用于修改现有的数据库对象。
    • 示例：ALTER TABLE table_name ADD COLUMN new_column INT;
  • DROP：用于删除数据库对象。
    • 示例：DROP TABLE table_name;

2.5.6 数据操作语言 (DML)

  • INSERT：用于向表中插入数据。
    • 示例：INSERT INTO table_name (column1, column2) VALUES (1, 'a');
  • SELECT：用于从表中查询数据。
    • 示例：SELECT * FROM table_name;
  • UPDATE：用于更新表中的数据。
    • 示例：UPDATE table_name SET column1 = 2 WHERE column2 = 'a';
  • DELETE：用于从表中删除数据。
    • 示例：DELETE FROM table_name WHERE column1 = 2;

2.5.7 其他关键字

  • AS：用于给列或表指定别名。
    • 示例：SELECT column1 AS col1 FROM table_name;
  • FROM：用于指定 SELECT 查询中的数据来源。
    • 示例：SELECT * FROM table_name;
  • WHERE：用于指定查询条件。
    • 示例：SELECT * FROM table_name WHERE column1 = 10;
  • GROUP BY：用于根据一个或多个列对结果集进行分组。
    • 示例：SELECT column1, COUNT(*) FROM table_name GROUP BY column1;
  • ORDER BY：用于对结果集进行排序。
    • 示例：SELECT * FROM table_name ORDER BY column1 ASC;
  • LIMIT：用于限制查询结果的数量。
    • 示例：SELECT * FROM table_name LIMIT 10;

这只是 ODPS SQL 关键字的一部分，实际使用时还有更多的关键字和功能需要了解。

2.5.8 集合操作

  • UNION：合并两个或多个 SELECT 语句的结果集，且不包含重复的行。
    • 示例：SELECT column1 FROM table1 UNION SELECT column1 FROM table2;
  • INTERSECT：返回两个 SELECT 语句结果集的交集，且不包含重复的行。
    • 示例：SELECT column1 FROM table1 INTERSECT SELECT column1 FROM table2;
  • EXCEPT：返回第一个 SELECT 语句的结果集中不在第二个 SELECT 语句结果集中的行。
    • 示例：SELECT column1 FROM table1 EXCEPT SELECT column1 FROM table2;

2.5.9 窗口函数

  • OVER：用于定义窗口函数的分区和排序规则。
    • 示例：SELECT column1, SUM(column2) OVER (PARTITION BY column1 ORDER BY column2) FROM table_name;

2.5.10 子查询

  • IN：用于判断某个值是否在子查询结果集中。
    • 示例：SELECT * FROM table1 WHERE column1 IN (SELECT column1 FROM table2);
  • EXISTS：用于判断子查询是否返回结果。
    • 示例：SELECT * FROM table1 WHERE EXISTS (SELECT * FROM table2 WHERE table1.column1 = table2.column1);

2.5.11 连接

  • JOIN：用于连接两个或多个表。
    • 示例：SELECT * FROM table1 JOIN table2 ON table1.column1 = table2.column1;
  • LEFT JOIN：左外连接，返回左表的所有行，即使右表中没有匹配的行。
    • 示例：SELECT * FROM table1 LEFT JOIN table2 ON table1.column1 = table2.column1;
  • RIGHT JOIN：右外连接，返回右表的所有行，即使左表中没有匹配的行。
    • 示例：SELECT * FROM table1 RIGHT JOIN table2 ON table1.column1 = table2.column1;
  • FULL JOIN：全外连接，返回左表和右表中所有的行，即使其中一个表中没有匹配的行。
    • 示例：SELECT * FROM table1 FULL JOIN table2 ON table1.column1 = table2.column1;

2.5.12 条件表达式

  • CASE：用于实现条件选择。
    • 示例：SELECT column1, CASE WHEN column1 > 0 THEN 'Positive' ELSE 'Non-Positive' END FROM table_name;

2.5.13 聚合函数

  • SUM：求和函数。
    • 示例：SELECT SUM(column1) FROM table_name;
  • COUNT：计数函数。
    • 示例：SELECT COUNT(*) FROM table_name;
  • AVG：平均值函数。
    • 示例：SELECT AVG(column1) FROM table_name;
  • MAX：最大值函数。
    • 示例：SELECT MAX(column1) FROM table_name;
  • MIN：最小值函数。
    • 示例：SELECT MIN(column1) FROM table_name;

2.5.14 数据导入导出

  • LOAD DATA：用于将数据从文件加载到表中。
    • 示例：LOAD DATA INPATH 'oss://path/to/file' INTO TABLE table_name;
  • INSERT OVERWRITE：用于将查询结果覆盖到表或分区中。
    • 示例：INSERT OVERWRITE TABLE table_name SELECT * FROM another_table;

2.5.15 事务控制

  • BEGIN：开始一个事务。
  • COMMIT：提交当前事务。
  • ROLLBACK：回滚当前事务。

2.5.16 权限管理

  • GRANT：用于授予用户或角色权限。
    • 示例：GRANT SELECT ON table_name TO user_name;
  • REVOKE：用于撤销用户或角色的权限。
    • 示例：REVOKE SELECT ON table_name FROM user_name;

2.5.17 系统管理

  • SHOW TABLES：用于显示数据库中的所有表。
  • DESCRIBE：用于显示表的结构信息。
    • 示例：DESCRIBE table_name;
  • EXPLAIN：用于显示 SQL 语句的执行计划。
    • 示例：EXPLAIN SELECT * FROM table_name;

这些关键字代表了 ODPS SQL 的基本功能和操作，但并不是全部。根据具体的需求和使用场景，可能还会涉及更多的关键字和高级功能。
