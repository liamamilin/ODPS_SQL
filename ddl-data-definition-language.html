ODPS SQL

Chapter 3 DDL (Data Definition Language)

ODPS SQL中的DDL（Data Definition Language）语句用于定义和管理数据库中的对象，如表、分区、视图等。本文将逐一介绍ODPS SQL中常用的DDL语句，包括表操作、分区和列操作、生命周期操作、视图操作、物化视图操作、参数化视图以及Schema操作，并提供相应的语法和示例。

3.1 表操作

创建表、修改表的所有权、修改表的保护属性、修改表的最后修改时间、修改表的生命周期、删除表、查看表的详细信息、获取表的元数据、获取表的分区信息以及列出所有表的语法和相关的使用场景。

3.1.1 创建表（Create Table）

语法

CREATE TABLE table_name (column_name data_type [comment 'column_comment'], ...) 
COMMENT 'table_comment' 
PARTITIONED BY (partition_column data_type) 
LIFECYCLE days;

示例：

CREATE TABLE employee (id BIGINT, name STRING COMMENT '员工姓名', age BIGINT COMMENT '员工年龄') 
COMMENT '员工信息表' 
PARTITIONED BY (dept STRING) 
LIFECYCLE 365;

3.1.2 修改表的所有权（Alter Table Owner）

语法：

ALTER TABLE table_name SET OWNER 'new_owner';

示例：

ALTER TABLE employee SET OWNER 'admin';

3.1.3 修改表的生命周期（Alter Table Lifecycle）

语法：

ALTER TABLE table_name SET LIFECYCLE days;

示例：

ALTER TABLE employee SET LIFECYCLE 730;

3.1.4 删除表（Drop Table）

语法：

DROP TABLE table_name;

示例：

DROP TABLE employee;

3.1.5 查看表的详细信息（Describe Table）

语法：

DESCRIBE table_name;

示例：

DESCRIBE employee;

3.1.6 获取表的元数据（List Table）

语法：

SHOW TABLE EXTENDED LIKE 'table_name' PARTITION(partition_description);

示例：

SHOW TABLE EXTENDED LIKE 'employee' PARTITION(dept='HR');

3.1.7 获取表的分区信息（List Partitions）

语法：

SHOW PARTITIONS table_name;

示例：

SHOW PARTITIONS employee;

3.1.8 修改表的注释

alter table sale_detail set comment 'new coments for table sale_detail';

3.1.9 重命名表

alter table sale_detail rename to sale_detail_rename;

3.2 分区和列操作

以下内容是关于MaxCompute分区操作的整理，包括新增分区、删除分区、修改分区、合并分区和清空分区数据的相关命令和使用示例。

3.2.1 新增分区

  • 限制条件：

    • 单表支持的分区数量上限为6万个。
    • 必须指明全部的分区。
    • 仅支持新增分区值，不支持新增分区字段。

  • 命令格式：

    alter table <table_name> add [if not exists] partition <pt_spec> [partition <pt_spec> partition <pt_spec>...];

  • 参数说明：

    • table_name：待新增分区的分区表名称。
    • if not exists：如果未指定且同名的分区已存在，会执行失败并返回报错。
    • pt_spec：新增的分区，格式为(partition_col1 = partition_col_value1, partition_col2 = partition_col_value2, ...)。

  • 使用示例：

    -- 示例1：添加一个分区
alter table sale_detail add if not exists partition (sale_date='201312', region='hangzhou');

-- 示例2：同时添加两个分区
alter table sale_detail add if not exists partition (sale_date='201312', region='beijing') partition (sale_date='201312', region='shanghai');

3.2.2 删除分区

  • 限制条件：

    • 每个分区过滤子句只能访问一个分区列。
    • 表达式用到的函数必须是内建的Scalar函数。

  • 命令格式：

    -- 未指定筛选条件
alter table <table_name> drop [if exists] partition <pt_spec>;
-- 指定筛选条件
alter table <table_name> drop [if exists] partition <partition_filtercondition>;

  • 参数说明：

    • table_name：待删除分区的分区表名称。
    • if exists：如果未指定且分区不存在，则返回报错。
    • pt_spec：删除的分区，格式为(partition_col1 = partition_col_value1, partition_col2 = partition_col_value2, ...)。
    • partition_filtercondition：分区筛选条件。

  • 使用示例：

    -- 示例：从表sale_detail中删除一个分区
alter table sale_detail drop if exists partition(sale_date='201312', region='hangzhou');

3.2.3 修改分区

  • 修改分区的更新时间：

    • 命令格式：alter table <table_name> touch partition (<pt_spec>);

    • 使用示例：

      -- 修改表sale_detail的分区sale_date='201312', region='shanghai'的LastModifiedTime
alter table sale_detail touch partition (sale_date='201312', region='shanghai');
  • 修改分区值：

    • 命令格式：alter table <table_name> partition (<pt_spec>) rename to partition (<new_pt_spec>);

    • 使用示例：

      -- 修改表sale_detail的分区值
alter table sale_detail partition (sale_date = '201312', region = 'hangzhou') rename to partition (sale_date = '201310', region = 'beijing');

3.2.4 合并分区

  • 使用限制：

    • 不支持外部表，聚簇表合并后的分区会消除聚簇属性。
    • 一次性合并分区数量限制为4000个。

  • 命令格式：

    alter table <table_name> merge [if exists] partition (<predicate>) [, partition(<predicate2>) ...] overwrite partition (<fullpartitionSpec>) [purge];

  • 参数说明：

    • table_name：待合并分区的分区表名称。
    • if exists：如果未指定且分区不存在，会执行失败并返回报错。
    • predicate：筛选待合并分区需要满足的条件。
    • fullpartitionSpec：目标分区信息。
    • purge：可选关键字。选择该字段，则会

清理session目录，默认清理3天内的日志。

  • 使用示例：

    -- 合并所有满足hh='00'的分区到hh='00'，mm='00'中
alter table intpstringstringstring merge partition(hh='00') overwrite partition(ds='20181101', hh='00', mm='00');

3.2.5 清空分区数据

  • 命令格式：

    -- 未指定筛选条件
truncate table <table_name> partition <pt_spec>[, partition <pt_spec>....];
-- 指定筛选条件
truncate table <table_name> partition <partition_filtercondition>;

  • 参数说明：

    • table_name：待清空分区数据的分区表名称。
    • pt_spec：待清空数据的分区，格式为(partition_col1 = partition_col_value1, partition_col2 = partition_col_value2, ...).
    • partition_filtercondition：分区筛选条件。

  • 使用示例：

    -- 从表sale_detail中清空一个分区
truncate table sale_detail partition(sale_date='201312', region='hangzhou');

  • 命令格式：

    alter table <table_name> change [column] <old_column_name> <new_column_name> <new_data_type>;

  • 使用示例：

3.2.9 修改列的顺序

  • 命令格式：

    alter table <table_name> change <old_column_name> <new_column_name> <column_type> after <column_name>;

  • 参数说明：

    • table_name：待修改列顺序的表名称。
    • old_column_name：待修改顺序的列的原始名称。
    • new_col_name：修改后的列名称。
    • column_type：待修改的列的原始数据类型。

  • 使用示例：

  • 参数说明：

3.3 生命周期操作

以下内容是关于MaxCompute表的生命周期设置的整理，包括生命周期的概念、设置生命周期的命令格式、参数说明以及使用示例。

3.3.1 生命周期概念

  • MaxCompute表的生命周期指的是表（分区）数据从最后一次更新的时间算起，在经过指定的时间后没有变动，则此表（分区）将被MaxCompute自动回收。
  • 对于非分区表，当LastModifiedTime超过设定的生命周期时长时，MaxCompute会回收这些数据并删除对应的表。
  • 对于分区表，根据各分区的LastModifiedTime判断该分区数据是否该被回收，当最后一个分区被回收后，该表不会被删除。

3.3.2 设置生命周期

3.3.2.1 创建表时设置生命周期

  • 命令格式：

    CREATE TABLE <table_name> (columns)
    partitioned BY (partition_columns)
    lifecycle <days>;

  • 参数说明：

    • table_name：需要设置生命周期的表名称。
    • days：生命周期时间，只能为正整数，单位为天。

  • 使用示例：

    CREATE TABLE test_lifecycle (key string) lifecycle 100;

3.3.2.2 修改已有表的生命周期

  • 命令格式：

    ALTER TABLE <table_name> SET lifecycle <days>;

  • 参数说明：

    • table_name：需要修改生命周期的表名称。
    • days：修改后的生命周期时间，只能为正整数，单位为天。

  • 使用示例：

    ALTER TABLE test_lifecycle SET lifecycle 50;

3.3.3 禁止或恢复生命周期

  • 命令格式：

    ALTER TABLE <table_name> [<pt_spec>] {enable|disable} lifecycle;

  • 参数说明：

    • table_name：待禁止或恢复生命周期的表的名称。
    • pt_spec：待禁止或恢复生命周期表的分区信息，对于有多级分区的表，必须指明全部的分区值。
    • enable：恢复表或分区的生命周期功能。
    • disable：禁止表或分区的生命周期功能。

  • 使用示例：

    -- 禁止表trans的生命周期功能
ALTER TABLE trans disable lifecycle;

-- 禁止表trans中时间为20141111分区的生命周期功能
ALTER TABLE trans partition (dt='20141111') disable lifecycle;

3.3.4 注意事项

  • 当项目级别与表级别的设置发生冲突，会优先使用表级别的设置。
  • 为分区表设置生命周期时，该生命周期设置将应用于整个表，包括所有的分区，不支持在分区级别单独设置或修改生命周期时长。
  • 数据被回收后，默认在回收站保留一天，超过一天，数据将被永久清理，清理后将无法恢复。
  • 生命周期主要提供定期回收表或分区的功能，每天根据服务的繁忙程度，不定时回收。不能确保表或分区的生命周期到期后，立刻被回收。
  • 在进行数据写入、更新和删除操作时，会更新LastModifiedTime；而修改生命周期时，不会影响LastModifiedTime。

3.4 视图操作

以下内容是关于MaxCompute视图操作的整理，包括创建或更新视图、重命名视图、查看视图、删除视图和修改视图的所有人的相关命令和使用示例。

3.4.1 视图操作命令

3.4.1.1 创建或更新视图

  • 命令格式：

    create [or replace] view [if not exists] <view_name>
    [(<col_name> [comment <col_comment>], ...)]
    [comment <view_comment>]
    as <select_statement>;

  • 参数说明：

    • or replace：可选。更新视图需要携带该字段。
    • if not exists：可选。如果视图已经存在时用create view会导致异常，可以用create or replace view重建视图。
    • view_name：必填。待创建或更新的视图的名称。
    • col_name：必填。待创建视图包含的列名称。
    • col_comment：可选。待创建视图的列的注释。
    • view_comment：可选。待创建视图的注释。
    • select_statement：必填。select查询语句，是视图的数据来源。

  • 使用示例：

    -- 基于表sale_detail创建视图sale_detail_view
create view if not exists sale_detail_view
(store_name, customer_id, price, sale_date, region)
comment 'a view for table sale_detail'
as select * from sale_detail;

-- 基于表sale_detail更新视图sale_detail_view
create or replace view if not exists sale_detail_view
(store_name, customer_id, price)
comment 'a view for table sale_detail'
as select shop_name, customer_id, total_price from sale_detail;

3.4.1.2 重命名视图

  • 命令格式：

    alter view <view_name> rename to <new_view_name>;

  • 参数说明：

    • view_name：必填。需要重命名视图的名称。
    • new_view_name：必填。新的视图名称。

  • 使用示例：

    -- 将视图sale_detail_view重命名为market
alter view sale_detail_view rename to market;

3.4.1.3 查看视图

  • 查看视图的命令详情请参见查看表或视图信息。

3.4.1.4 删除视图

  • 命令格式：

    drop view [if exists] <view_name>;

  • 参数说明：

    • if exists：可选。如果视图不存在且没有指定if exists，会返回报错。
    • view_name：必填。待删除的视图的名称。

  • 使用示例：

    -- 删除视图sale_detail_view
drop view if exists sale_detail_view;

3.4.1.5 修改视图的所有人

  • 命令格式：

    alter view <view_name> changeowner to <new_owner>;

  • 参数说明：

    • view_name：必填。待修改Owner的视图的名称。
    • new_owner：必填。修改后的Owner的账号。

  • 使用示例：

    -- 将视图sale_detail_view的所有人修改为ALIYUN$xxx@aliyun.com
alter view sale_detail_view changeowner to 'ALIYUN$xxx@aliyun.com';
